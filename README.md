# Godot-Wild-Jam_03-2026
## Player Mechanics:
Player can get older or younger, depending on how much time they have currently. There are tradeoffs for being old and young. You have less time when young, but you are smaller, which could let you pass certain barriers, and you're slower/have slightly less mobility when old.

## Timer:
Better to have a bigger progress bar in the top centre of the screen than a label

## Level Design:
The levels should be maybe about 4x the width of the screen, camera following the player
Each level associated with a mechanic

First level: start with not that much time, have the tree growing next to them, make the player take the time from the tree to make it get smaller. Since the player is smaller when little time left, have a little tunnel that lets them go the rest of the way

Second level: Closed door(vertical platform), see it going down once, knows how it will move. Also add simple traps.
